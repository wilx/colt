Changes and Bugs in MIF Doclet

Last modified: Fri Jul 23 15:27:04 PDT 1999

MIF Doclet Home Page

Limitations

This doclet does not support converting HTML tables to MIF. We have not committed to adding table support. See other limitations.

Changes and Bugs

You can tell which version of the MIF doclet you have only by file date and size, as listed on this page.

Version C (mifdoclet_C.zip) 

    mif.jar              84135 Aug 30 16:34
    mifoutput.jar        36075 Aug 30 16:22
    standarddoclet.jar  162636 Aug 31 16:34
  • Design Change: The file standarddoclet.jar is now included by necessity, since some changes in it are required by the MIF Doclet. This means the MIF doclet can work with Java 2 SDK v1.2 and no longer requires v1.2.2. The MIF Doclet also works with v1.3 Beta. Eventually the MIF Doclet will work with tools.jar in v1.3 Final without requiring this separate jar file.
  • Design Change: Changed the name of the jar files: mif.jar became mifoutput.jar and mifdoclet.jar became mif.jar.
  • Design change: Changed the name of the package in which the MIF Doclet code resides. As a result, the -doclet option now requires com.sun.tools.doclets.mif.MIFDoclet as an argument.
  • Added Feature: Renamed -fmbatch to -batch. It now takes one of two arguments: fmbatch or dzbatcher.
  • Added Feature: Added -noexecutebatch option.
  • Added Feature: Alphabetically sort rows in class summary (on package page) and member summary (on class page)
  • Fixed Bug: The bug was that {@link name label} tags displayed the name rather than the label. The fix is to omit all HTML tags in the label and display the text that's left.
  • Fixed Bug: Parameter names are now included in the Constructor/Method Descriptions.
  • Fixed Bug: Fixed bug about @exception tags. Those were getting repeated.
  • Fixed Bug: Fixed "missing space" problem after first word in the next paragraph after </p> and </pre> tags.
  • Fixed Bug: Added escape character ('\') to any array square brackets [] that appear in Marker Text, like this: '\[' and '\]'.
  • Fixed Bug: The stylemap.properties file now can have blank lines
  • Fixed Bug: Added PackagePreHead to stylemap.properties. Renamed ClassPackageHead to ClassPreHead.
  • Fixed Bug: Fixed template.mif so it would no longer display an "unavailable font" warning when opened in Solaris.
  • Known Bug: TrueType fonts in the header/footer are corrupted with Distiller 3.0. (Workaround: Change fonts or use Distiller 4.0 instead.)
  • Known Bug: The paragraph ahead of tables is Paragraph style; should be ClassText style.
  • Known Bug: When converted to PDF, links in the index appear in the white space to the right of the page number.
Version B (mifdoclet_B.zip) 
    mif.jar        31977 Jul 20 16:06   (unchanged from version A)
    mifdoclet.jar  93524 Jul 29 11:43
  • Added Feature: Added -includeDir for specifying the directory of the three MIF include files.
  • Change: Removed -contentstitle and -indextitle options, which are obsoleted by -contentsfile and -indexfile.
  • Fixed Bug: The bug was that the references to .mif files should instead be to .fm files.
  • Fixed Bug: The bug was that a space was missing between words. This occured at the boundary of newlines in doc comments.
  • Fixed Bug: The bug was that OutlineHead style was white font; should be black.
  • Fixed Bug: The bug was that Array brackets [] did not appear in the return type.
  • Known Bug: When converted to PDF, links in the index appear in the white space to the right of the page number.
  • Known Bug: {@link name label} tags display the name rather than the label.
  • Known Bug: TrueType fonts in the header/footer are corrupted with Distiller 3.0. (Workaround: Change fonts or use Distiller 4.0 instead.)
Version A (mifdoclet_A.zip) 

   mif.jar         31977 Jul 20  16:19 
   mifdoclet.jar   98971 Jul 20  16:19
  • Known Bug: A space is missing between words. This occurs at the boundary of newlines in doc comments.
  • Known Bug: When converted to PDF, links in the index appear in the white space to the right of the page number.
  • Known Bug: OutlineHead style is white font; should be black.
  • Known Bug: titlepage.mif was inadvertently omitted from the download bundle. You can get it from the MIF doclet home page. NOTE: This file contains a dummy copyright information on page 2. Be sure to modify this for your company.
  • Known Bug: There are some content bugs not present in the standard doclet. Array brackets [] do not appear in the return type.
  • Known Bug: {@link name label} tags display the name rather than the label.
  • Known Bug: In Solaris, opening template and any generated MIF files produces warning that "Arial", "TimesNewRoman" and "TimesNewRoman Bold" fonts are not available.
